# 球队球员采集画像 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build the team/player profile module with staged collection review, official profile facts, backend APIs, frontend profile centers, OpenSpec validation, and code review evidence.

**Architecture:** Add profile-specific tables on top of the existing core data schema. Use Spring Boot Java 17 + JdbcTemplate for weak-structure fact aggregation and Vue 3 + Element Plus for admin UI.

**Tech Stack:** Java 17, Spring Boot 3.3, Flyway, H2 tests/MySQL runtime, Vue 3, TypeScript, Element Plus, Vitest, OpenSpec.

---

## File Structure

- Create `server/src/main/resources/db/migration/V4__create_profile_tables.sql`: collection and profile fact tables.
- Create `server/src/main/java/com/worldcup/profile/api/ProfileController.java`: REST endpoints.
- Create `server/src/main/java/com/worldcup/profile/api/dto/ProfileDtos.java`: request/response records.
- Create `server/src/main/java/com/worldcup/profile/service/ProfileCollectionService.java`: collection approval/rejection.
- Create `server/src/main/java/com/worldcup/profile/service/ProfileQueryService.java`: team/player profile queries.
- Create `server/src/test/java/com/worldcup/profile/api/ProfileControllerTest.java`: authenticated API tests.
- Create `server/src/test/java/com/worldcup/profile/service/ProfileCollectionServiceTest.java`: approval/idempotency tests.
- Create `client/src/api/profiles.ts`: frontend API helper.
- Create `client/src/views/TeamProfilesView.vue`: team profile center.
- Create `client/src/views/PlayerProfilesView.vue`: player profile center.
- Modify `client/src/router/index.ts`: profile routes.
- Modify `client/src/views/DashboardView.vue`: real profile links.
- Create/modify frontend tests in `client/src/__tests__`.
