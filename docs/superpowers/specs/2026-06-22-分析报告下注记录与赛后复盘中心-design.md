# 分析报告下注记录与赛后复盘中心设计

日期：2026-06-22（北京时间）
状态：已按用户目标进入 OpenSpec 文档阶段

## 背景

项目已完成 Java 前后端骨架、JSON 审核入库、核心数据模型、球队/球员画像、比赛中心、赔率中心、舆情与外部因素中心。下一阶段要把 AI/Claude/Codex 生成的分析报告、下注方式、实际下注记录和赛后复盘纳入正式管理。

用户补充规则：下注方式也由 AI 分析后生成 JSON，用户批准后保存到数据库。Java 系统不直接生成下注方式。

## 目标

构建一个个人使用的“分析报告 + AI 下注方案 + 实际下注记录 + 赛后复盘”工作台：

- 批准后的 ANALYSIS JSON 保存分析报告、AI 下注方案和赛后复盘。
- 批准后的 BETS JSON 保存实际出票、结算、CLV 和盈亏。
- 前端清晰区分 AI 下注方案与实际出票记录。
- 提供 ROI、CLV、总投入、总返还、净盈亏和复盘数量统计。
- 保留每条数据的原始 JSON，方便追溯。

## 非目标

- 不生成新分析，不生成新下注方式。
- 不接入外部 AI、新闻、赔率或彩票平台。
- 不修改 `CLAUDE.md`、`skill/` 规则与档案。
- 不做复杂可视化图表；图表留给后续完整工作台阶段。

## 数据流

```text
AI/Claude/Codex 生成 JSON
  ↓
用户在 JSON 审核中心批准
  ↓
BusinessJsonMapper 入库
  ↓
analysisreviewcenter API 查询
  ↓
/analysis-review 前端展示与统计
```

## 统计口径

- 总投入 = `bets.stake` 求和。
- 总返还 = `bets.return_amount` 求和。
- 净盈亏 = `bets.profit_loss` 求和。
- ROI = 净盈亏 / 总投入，总投入为 0 时为 0。
- 平均 CLV = 非空 `bets.clv` 平均值。

## 安全与边界

- 所有 API 需要认证。
- 页面文案必须强调“AI 方案归档”和“实际出票记录”分离。
- 不新增“推荐下注”“加注”“倍投”“稳赚”“必胜”等 Java 推导文案。
- AI 方案中的金额建议可以展示，但必须来自已批准原始 JSON。
