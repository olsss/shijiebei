# 赔率中心与盘口快照设计

## 背景

当前 Java 系统已经具备 JSON 审核入库、核心比赛数据、球队/球员画像和比赛中心。已有 `odds_snapshots` 表可以保存赔率原始快照，但结构只有 `bookmaker + market_type + odds_value + raw_payload`，无法完整表达一个玩法下的多个选项赔率。例如：

- 胜平负：主胜 / 平 / 客胜。
- 让球胜平负：让胜 / 让平 / 让负，并需要保存让球数。
- 比分：0-0、1-0、1-1、2-1、其它等多个比分选项。
- 总进球：0、1、2、3、4、5、6、7+。
- 半全场：胜胜、胜平、胜负、平胜、平平、平负、负胜、负平、负负。
- 亚洲让球、大小球、其它海外盘口：需要保存盘口线和每个选择的赔率。

用户明确要求“各个玩法的赔率都保存”。因此本阶段必须把赔率从单值快照扩展为“快照批次 → 玩法市场 → 选项赔率”的结构，并提供赔率中心页面查看。

## 目标

1. 新增结构化赔率表，保存每个比赛、公司、玩法、盘口线、快照类型和抓取时间。
2. 保存每个玩法下所有选项赔率，不能只保存一个 `odds_value`。
3. 改造 JSON 正式入库流程，使赔率 JSON 批准后同时写入：
   - 兼容旧概览的 `odds_snapshots`；
   - 新增 `odds_market_snapshots`；
   - 新增 `odds_selection_snapshots`。
4. 新增赔率中心后端 API，支持按比赛、公司、玩法查看赔率市场与选项。
5. 新增 `/odds` 前端页面，展示赔率总览、筛选、玩法市场和选项赔率。
6. 只做事实数据保存与展示，不做下注推荐、不做资金分配、不输出投注结论。
7. 完成 OpenSpec 设计审查和代码审查报告。

## 非目标

- 不调用外部赔率 API，不新增密钥配置。
- 不计算投注建议、下注金额或推荐组合。
- 不替代 `skill/` 的比赛分析流程。
- 不修改 `CLAUDE.md`、`skill/SKILL.md`、`skill/rules/` 或 `skill/archive/`。
- 不做历史赔率曲线图的高级可视化；本阶段以表格与详情为主。

## 数据模型

### `odds_market_snapshots`

保存一条“某比赛、某公司、某玩法、某盘口线、某时间点”的市场快照。

字段：

- `id`
- `import_item_id`：来源审核项。
- `odds_snapshot_id`：兼容旧表的一条父快照，可为空。
- `match_id`
- `bookmaker`
- `market_code`：规范代码，如 `HAD`、`HHAD`、`CORRECT_SCORE`、`TTG`、`HAFU`、`ASIAN_HANDICAP`、`OVER_UNDER`、`RAW`。
- `market_name`：中文/来源玩法名。
- `snapshot_type`：`OPEN`、`LIVE`、`CLOSING`、`RAW`。
- `handicap_line`：数字盘口线，如让球 -1、大小球 2.5。
- `line_value`：来源原始盘口字符串。
- `captured_at`
- `source_ref`
- `raw_payload`
- `created_at`

### `odds_selection_snapshots`

保存某个市场快照下的每个选项赔率。

字段：

- `id`
- `market_snapshot_id`
- `selection_code`：规范代码，如 `HOME`、`DRAW`、`AWAY`、`HANDICAP_HOME`、`1-0`、`7+`、`DRAW_HOME`。
- `selection_name`：展示名，如 主胜、平、客胜、让胜、比分 1-0。
- `odds_value`
- `implied_probability`：可为空。
- `selection_status`：`OPEN`、`SUSPENDED`、`CLOSED`、`UNKNOWN`。
- `raw_payload`
- `created_at`

## JSON 入库兼容策略

`BusinessJsonMapper.mapOdds()` 继续支持现有 JSON 格式，同时增强解析：

- 顶层 `companies[]` / `all_books[]` / `markets[]`。
- 公司下 `markets[]`。
- 市场下 `selections[]` / `outcomes[]` / `options[]`。
- 市场下 `odds` 或 `prices` 对象，例如 `{ "HOME": 1.8, "DRAW": 3.4, "AWAY": 4.2 }`。
- 单市场单赔率旧格式仍写入一个 `RAW` 选项，避免兼容断裂。

映射规则：

- `market`、`market_type`、`type`、`key`、`code` 映射到 `market_code`。
- `name`、`market_name`、`玩法` 映射到 `market_name`。
- `snapshot_type`、`phase` 映射到 `snapshot_type`，无法识别时为 `RAW`。
- `handicap`、`handicap_line`、`line` 映射盘口线。
- 每个 selection 的 `code`、`name`、`selection`、`label`、`odds`、`price`、`value` 被结构化保存。

## 后端 API

所有接口需要认证，统一 `ApiResponse<T>`。

- `GET /api/odds`
  - 返回赔率中心总览列表，按比赛/公司/玩法聚合。
- `GET /api/odds/matches/{matchId}`
  - 返回某比赛的完整赔率详情，含市场与全部选项。
- `GET /api/odds/bookmakers`
  - 返回已入库公司列表。
- `GET /api/odds/markets`
  - 返回已入库玩法列表。

新增：

- 路由 `/odds`
- Dashboard 模块卡片“赔率中心”

页面展示：

- 顶部统计：比赛数、公司数、玩法数、选项赔率数。
- 左侧/上方筛选：公司、玩法。
- 市场表：比赛、公司、玩法、盘口线、快照类型、抓取时间、选项数量。
- 详情表：当前市场的全部选项赔率。
