# 舆情与外部因素中心设计

## 背景

当前 Java 系统已经具备 JSON 审核入库、核心比赛数据、球队/球员画像、比赛中心和赔率中心。比赛分析规则中明确要求关注天气、场地、裁判、赛程体能、战意、轮换、默契球风险、队内矛盾、主帅压力、媒体观点、球迷情绪和市场热门程度等因素。但当前系统只有 `matches.external_factors` 这样的粗粒度字段，无法把每条事实、观点、来源、可信度、时效和风险评分结构化保存。

用户确认本阶段采用 A1 + A2：

- A1：事实型舆情与外部因素中心，保存可核查记录和来源。
- A2：风险评分中心，保存人工/AI 给出的风险类型、风险等级和风险分，但不自动产出比赛结论或下注建议。

## 目标

1. 批准后的 SOURCE JSON 可以结构化入库舆情与外部因素事实记录。
2. 保存每条记录的比赛、分类、标题、摘要、影响方向、来源、可信度、置信度、抓取时间、有效期和原始 JSON。
3. 保存风险评分：战意风险、舆论过热、内部不稳、伤病被低估、数据过期、默契球风险等。
4. 查询接口支持总览、按比赛详情、分类字典和风险类型字典。
5. 前端新增 `/sentiment` 页面，展示事实记录、风险评分、过期提醒和原始 payload。
6. 不修改 `CLAUDE.md` 与 `skill/`，不让 Java 系统自动进行比赛判断。
7. OpenSpec 文档完成后先做文档审查；代码完成后再做代码审查。

## 非目标

- 不抓取外部新闻、社媒或天气 API。
- 不自动判断比赛胜平负、比分或投注价值。
- 不生成下注金额、推荐组合或稳胆结论。
- 不修改比赛分析 skill 流程。
- 不做复杂趋势图；本阶段以结构化表格、详情和风险摘要为主。

## 数据模型

### `match_context_factors`

保存 A1 事实型记录。每一行代表某场比赛的一条外部因素或舆情记录。

字段：

- `id`
- `import_item_id`：来源审核项。
- `match_id`
- `factor_category`：大类，如 `WEATHER`、`VENUE`、`REFEREE`、`TRAVEL`、`REST`、`MOTIVATION`、`ROTATION`、`LOCKER_ROOM`、`MEDIA`、`PUBLIC_SENTIMENT`、`MARKET_HEAT`、`OTHER`。
- `factor_type`：来源细分类型，如 `RAIN`、`HIGH_TEMPERATURE`、`STRICT_REFEREE`、`MUST_WIN`、`RUMOR`。
- `title`
- `summary`
- `impact_direction`：`POSITIVE`、`NEGATIVE`、`NEUTRAL`、`MIXED`、`UNKNOWN`。
- `entity_type`：`MATCH`、`TEAM`、`PLAYER`、`COACH`、`REFEREE`、`OTHER`。
- `entity_key`：来源中的实体标识或名称。
- `evidence_level`：`OFFICIAL`、`DATA_VENDOR`、`MAIN_MEDIA`、`JOURNALIST`、`SOCIAL`、`RUMOR`、`UNKNOWN`。
- `source_name`
- `source_url`
- `source_ref`
- `observed_at`
- `expires_at`：超过即视为时效风险。
- `confidence_score`
- `reliability_score`
- `raw_payload`
- `created_at`
- `updated_at`

### `sentiment_risk_assessments`

保存 A2 风险评分。风险可关联一条 factor，也可以直接关联比赛。

字段：

- `id`
- `import_item_id`
- `match_id`
- `factor_id`
- `risk_type`：如 `MOTIVATION_RISK`、`PUBLIC_OVERHEAT`、`LOCKER_ROOM_RISK`、`INJURY_UNDERESTIMATED`、`DATA_STALE`、`COLLUSION_RISK`。
- `risk_level`：`LOW`、`MEDIUM`、`HIGH`、`CRITICAL`。
- `risk_score`：0-100。
- `title`
- `rationale`
- `suggested_action`：只允许风险处理动作，如 `VERIFY_AGAIN`、`LOWER_CONFIDENCE`、`MONITOR`、`NO_ACTION`；不能写下注建议。
- `source_name`
- `source_ref`
- `raw_payload`
- `created_at`

## JSON 入库策略

继续使用现有 `SOURCE` 类型，不新增 ImportItemType。`BusinessJsonMapper.mapSource()` 在保留 `source_evidence`、`data_conflicts`、`aliases` 行为的基础上增加解析：

- `external_factors[]` / `factors[]` → `match_context_factors`。
- `sentiment_records[]` / `sentiments[]` → `match_context_factors`。
- 单个 `weather`、`venue`、`referee`、`motivation`、`public_sentiment` 对象可转换为 factor。
- factor 内的 `risks[]` → `sentiment_risk_assessments`。
- 顶层 `risk_assessments[]` / `risks[]` → `sentiment_risk_assessments`。

示例：

```json
{
  "match": "德国 vs 日本",
  "date": "2026-06-24",
  "jc_code": "周三001",
  "external_factors": [
    {
      "category": "WEATHER",
      "type": "RAIN",
      "title": "预计小雨",
      "summary": "赛前 2 小时有小雨，草皮可能偏滑。",
      "impact_direction": "MIXED",
      "evidence_level": "DATA_VENDOR",
      "source_name": "Weather Provider",
      "reliability": "8.0",
      "expires_at": "2026-06-24T19:00:00"
    }
  ],
  "sentiment_records": [
    {
      "category": "PUBLIC_SENTIMENT",
      "title": "热门方舆论过热",
      "summary": "主流预测集中支持德国大胜。",
      "impact_direction": "NEGATIVE",
      "evidence_level": "MAIN_MEDIA",
      "source_name": "Media Digest",
      "risks": [{"type":"PUBLIC_OVERHEAT","level":"HIGH","score":"78","rationale":"热门预期过度集中"}]
    }
  ]
}
```

## 后端 API

所有接口需要认证，统一 `ApiResponse<T>`。

- `GET /api/sentiment`
  - 返回舆情与外部因素总览，包含比赛、分类、标题、影响方向、来源、可信度、置信度、是否过期、风险数量和最高风险等级。
- `GET /api/sentiment/matches/{matchId}`
  - 返回某场比赛的全部 factors 和 risk assessments。
- `GET /api/sentiment/categories`
  - 返回已入库 factor 分类。
- `GET /api/sentiment/risk-types`
  - 返回已入库风险类型。

## 前端设计

新增：

- `client/src/api/sentiment.ts`
- `client/src/views/SentimentCenterView.vue`
- 路由 `/sentiment`
- Dashboard 模块卡片“舆情与外部因素”

页面展示：

- 顶部统计：比赛数、因素记录数、风险评分数、过期记录数、高风险数。
- 筛选：分类、风险类型、是否过期。
- 左侧/主表：外部因素与舆情记录。
- 右侧详情：当前比赛 factors、风险评分、来源、原始 JSON。
- 风险标签：`LOW/MEDIUM/HIGH/CRITICAL` 显示不同颜色。

## 风险评分边界

A2 只保存风险评分，不生成比赛推荐。

允许：

- “战意风险 HIGH，建议赛前再核。”
- “舆论过热 78 分，建议降低置信度。”
- “伤病信息过期，建议补采。”

禁止：

- “推荐主胜。”
- “建议买让负。”
- “投入 100 元。”
- “稳胆。”
