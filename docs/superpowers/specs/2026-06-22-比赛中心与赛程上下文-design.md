# 比赛中心与赛程上下文设计

## 背景

当前 Java 系统已经具备：

- JSON 审核入库：外部/AI 生成 JSON 必须先暂存、审核，批准后才进入正式业务表。
- 核心数据模型：球队、球员、比赛、阵容、事件、球队统计、球员统计、证据链、冲突、赔率、报告、下注记录等表已落地。
- 球队/球员画像：可查看球队特性、球员状态、伤病、红黄牌、更衣室、进球时间点和外部因素。

下一步需要把 `matches` 及其上下文做成一个独立的比赛中心，供本人快速查看赛程、赛前/赛中/赛后上下文、阵容、事件、统计、证据和冲突。该模块只呈现正式业务库数据，不参与比赛分析结论，不直接给下注建议。

## 目标

1. 后端新增比赛中心查询 API：比赛列表、比赛详情、阵容、事件、球队统计、球员统计。
2. 比赛列表能够作为赛程中心使用，展示比赛日、开球时间、竞彩编号、赛事、阶段、场地、主客队和数据完整度计数。
3. 比赛详情聚合阵容/首发、事件、球队统计、球员统计、外部因素、证据链和冲突。
4. 前端新增 `/matches` 页面，提供比赛列表、详情看板、阵容/事件/统计/证据冲突分区。
5. OpenSpec change 使用中文命名，并完成严格校验。
6. 代码完成后执行完整验证，并由子代理/审查流程输出审查记录。

## 非目标

- 不新增自动爬虫、外部付费接口调用或密钥管理。
- 不生成“推荐投注”“下注金额”“比分建议”等分析结论。
- 不修改 `CLAUDE.md`、`skill/SKILL.md`、`skill/rules/` 或 `skill/archive/`。
- 不做比赛、球队、球员的全量 CRUD；本阶段只查询已批准/已入库的数据。
- 不改动赔率中心、分析报告中心和赛后复盘中心；这些作为后续阶段。

## 数据来源与边界

本阶段直接复用既有 V3/V4 表：

- `matches`：赛程和基础上下文，含 `external_factors`。
- `match_lineups`：首发、替补、上阵人员。
- `match_events`：进球、红黄牌、换人等事件。
- `match_team_stats`：球队维度进失球、首球分钟、进球分钟、统计 payload。
- `match_player_stats`：球员分钟、进球、助攻、红黄牌、统计 payload。
- `source_evidence`：与比赛相关的证据。
- `data_conflicts`：与比赛相关的数据冲突。
- `teams` / `players`：用于补齐展示名称。

不新增迁移表，避免过度设计。若后续需要“赛程上下文事实表”或“比赛标签”，应作为新的 OpenSpec change。

## 后端 API 设计

所有接口均需认证，统一返回 `ApiResponse<T>`。

- `GET /api/matches`
  - 返回 `MatchSummaryResponse[]`。
  - 排序：比赛日降序、开球时间降序、ID 降序。
  - 字段包含比赛基础信息、主客队名称、事件/阵容/证据/冲突计数。

- `GET /api/matches/{matchId}`
  - 返回 `MatchDetailResponse`。
  - 聚合 summary、lineups、events、teamStats、playerStats、evidence、conflicts、externalFactors。
  - 不存在返回 404。

- `GET /api/matches/{matchId}/lineups`
  - 返回阵容与首发/替补信息。

- `GET /api/matches/{matchId}/events`
  - 返回按分钟排序的事件。

- `GET /api/matches/{matchId}/team-stats`
  - 返回主客队或关联球队统计。

- `GET /api/matches/{matchId}/player-stats`
  - 返回球员技术统计，并补齐球队名。

## 前端设计

新增：

- `client/src/api/matches.ts`
  - 类型定义。
  - path builder：`buildMatchDetailPath`、`buildMatchLineupsPath` 等。
  - API helper：`listMatches`、`getMatchDetail`、`listMatchLineups`、`listMatchEvents`、`listMatchTeamStats`、`listMatchPlayerStats`。

- `client/src/views/MatchCenterView.vue`
  - 页面标题：比赛中心。
  - 顶部统计：比赛数、事件数、阵容条目、待处理冲突。
  - 左侧比赛列表：比赛日、竞彩编号、主客队、阶段、状态。
  - 右侧详情：基础上下文、外部因素、阵容、事件、球队统计、球员统计、证据链、冲突。

- `client/src/router/index.ts`
  - 增加 `/matches` route。

- `client/src/views/DashboardView.vue`
  - 增加/替换模块卡片“比赛中心”，指向 `/matches`。
