# 证据加权与置信度降级（对应阶段 4）

> 借鉴 FIFA-WINNER-SKILL 的证据加权（60/40），竞彩化改写：权重含义换成本项目维度。

## 加权框架
- **主权重（基本面 + 盘口）**：§2 维度 1–5（交锋/近况、阵容状态、教练战术、小组形势战意、赔率与庄家视角）。这是下注主依据。
- **辅权重（大赛气质 + 逐场 + 临场）**：§2 维度 6–9 + §2A + §2B。用于修正主权重，尤其防"牌面强但破密集差"的陷阱。
- 主辅冲突时（如盘口看好但大赛气质看衰）→ 按 §2A 末条**降低投入**，并加平/负或低比分保险。

## 置信度降级触发器
满足任一即降级（降级 = 减少该逻辑投入 / 增加保险方向 / 必要时放弃）：
- 缺阵容、伤停、首发数据（player-status 未当日刷新）；
- **数据过期（§14 时效阈值）**：首发/伤停/临场 >24h、球队档案或 group-rank 未含最新一轮或距今 >7 天、赔率 live 距停售 >3h——每个数据文件应标“最后更新时间”，过期即降级或补采；
- 来源核验状态为 `partial` 或 `blocked`（见 source-snapshot.md）；
- 热门方/必须赢但破密集能力存疑；
- 本届逐场样本显示慢热/终结差/守领先强（低比分倾向）；
- 命中行为偏差自检（§13：近因/光环、牌面、大众盘从众、锚定、结果偏差）；
- 单一逻辑投入逼近本轮给定胜负资金 60%–70% 上限。

## 结论标注（强制）
每场结论必须标：**实力判断 / 盘口判断 / 大赛气质判断**，并写明置信度（高/中/低）与触发的降级项。写入 `bets.json` 的 `结论类型` 与 `证据状态`。
