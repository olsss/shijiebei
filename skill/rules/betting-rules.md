# 竞彩下注核心规则镜像（CLAUDE.md §1–§15）

> 本文件镜像 `CLAUDE.md` 的下注硬规则，供 skill 各阶段引用；工具说明以 `CLAUDE.md` §10 与 `skill/SKILL.md` 资源索引为准。规则冲突时以根目录 `CLAUDE.md` 为准。

## §1 官方核对优先
- 每轮下注前，先核对中国体彩/竞彩官方计算器的比赛编号、开赛时间、玩法、让球数、是否开售。
- 不凭记忆写盘口；普通胜平负（HAD）和让球胜平负（HHAD）必须分开记录。
- 出票后以票面 SP 为准；网页/接口赔率只作为出票前估算。

## §2 分析框架（每场至少看这些维度）
1. 历史交锋与近期战绩；
2. 阵容配置、主力状态、伤停/停赛；
3. 教练状态、战术风格、轮换可能；
4. 小组形势、战意、赛程体能、天气/场地；
5. 赔率与庄家视角：以世界大公司赔率为主（Pinnacle 作标尺/凯利基准、William Hill、Betfair 等），看初赔→即时变化；体彩赔率去水重，仅作出票/结算的票面 SP，不作分析主依据；
6. 世界杯/大赛履历：近几届世界杯、洲际杯、预选赛和大赛淘汰/小组赛表现，不能只看纸面阵容；
7. 球队比赛气质：是否擅长低位防守、反击、定位球、守领先、拖节奏、身体对抗；
8. 破密集能力：强队/热门方是否有稳定中锋、边路爆点、倒三角、二点包抄和高质量定位球；控球多不等于有效进攻；
9. 冷门路径：平局、小胜、卡让球线、最后阶段失球、早早丢球后急躁、11打10仍打不开局面。

## §3 资金规则（动态预算）
- **预算不固定**：每轮下注前由本人给出“本轮可用资金总额”（未给出先问，不沿用旧的 100/50 默认值臆测）。所有金额按本轮给定总额的比例执行。
- 给定总额内分两类盘：胜平负/让球 一类，比分/总进球/半全场 一类（§12）；比例按本场判断分配，未指定时默认胜负类为主、低分玩法类为辅。
- 单一逻辑投入 ≤ 本轮胜负类资金的 60%–70%。
- 凯利只作**价值信号与相对注额权重**，非绝对注额：注额 = 给定预算内按各注凯利相对强弱分配 + 单一逻辑上限约束；建议分数凯利（1/4–1/2）防过注；Betfair 的“凯利>1”是结构性非价值不据此加注（见 odds-collection.md）。
- 价值不足/无正期望场次，**空仓也是合法决策**。
- 4 串 1 激进但容错低，必须接受全黑风险；不要追损、不要临场加倍。
- 让球深盘必须留保险：让胜、让平、让负至少考虑其中两个方向。
- 比分单关避免单压一个比分；主比分 + 卡线比分 + 冷门比分分层。

## §4 记录规则
- 记录一律 JSON 留档（**旧 Excel 已弃用**）：下注记录 `archive/bets.json`、每场分析 `archive/analysis/<id>.json`；比分/赛果以字符串存，天然防日期化。查看用 `archive/dashboard/index.html`（build_dashboard.py 生成）。
- 每张票记录：下注日期、**比赛日（统一用北京时间日期；外部赔率/赛程接口的 UTC 日期只作原始字段保存）**、类型（HAD/HHAD/比分/总进球/半全场/串关，§12）、票号、比赛、下注内容、投入、赔率/总赔率、**收盘赔率、CLV**、赛果、命中、返还、盈亏、备注。
- **CLV（收线价值）**：记下注入场赔率 + 该场临场收盘（停售前 live 快照）赔率，CLV = 入场赔率 ÷ 收盘赔率 − 1；结算用 `settle.py --closing-odds` 自动算（§5 赔率价值层）。
- 未开奖先写"待赛果 / 待开奖"，返还和盈亏留空；开奖后再结算。
- 若赔率与出票票面不同，备注写"官网估算，票面 SP 优先"。

## §5 复盘规则（赛后分层）
- 数学层：哪张票命中、总返还、净盈亏、ROI；
- 足球层：判断对/错的原因，特别是阵容、伤停、战意、教练与临场；
- 盘口层：是否被让球线卡住，是否低估平局/小胜/冷门；
- 大赛气质层：是否低估球队世界杯/大赛经验、破密集能力、守领先能力、身体对抗、低比分韧性；
- 赔率价值层（CLV）：逐票比入场赔率 vs 临场收盘赔率，正 CLV=抓到价值（输了也是好下注）、负 CLV=逆势；**短期 ROI 是方差，CLV 才是长期水平标尺**；
- 规则沉淀：每轮至少记录 1 条"下次必须避免/必须增加"的下注规则。

## §6 子代理审查规则
最终出票前若需审查，分类审查：
1. 数学/规则代理：玩法、让球数、赔率乘积、金额合计、结算口径（90 分钟/§11）、玩法是否开售与封顶（§12）；
2. 足球逻辑代理：阵容、伤病、战意、盘口风险、行为偏差自检（§13）；
3. 资金分配代理：是否过度集中、是否有必要加保险票、是否按本轮给定预算与凯利权重分配（§3）。

额外（条件触发）：
4. 大赛气质代理：热门方/深盘/必须赢/历史气质争议时必启；
5. 本届逐场表现代理：本届已开赛且球队已有样本时必启；
6. 数据源核查代理：涉赛果/技术统计/伤停/首发/临场时必启，≥2 独立来源并标官方/媒体/预测。

## §7 本轮特别规则（2026-06-20 四场，案例样本）
- 031 巴西 vs 海地：普通胜平负未开，只能买让球胜平负，巴西让 2 球。
- 巴西 -2：让胜=巴西赢 3 球或以上；让平=巴西正好赢 2 球；让负=巴西赢 0/1 球、打平或输球。
- 已出票第一张 20 元按票面 SP 结算，不再按网页变赔重算。

## §8 风险边界（本 skill 底线声明）
竞彩只能小额娱乐，不保证盈利；连续判断失误时减少投入而不是加注追回。

## §9 已沉淀复盘规则（2026-06-20）
- 土耳其 0-1 巴拉圭暴露问题：土耳其 11 打 10 仍缺少有效进攻，赛前高估了牌面和战意，低估了巴拉圭低位防守韧性。
- 以后遇到"热门方必须赢 + 对手能低位死守"的比赛，不能重压热门胜；必须提高平/负保险。
- 比分单关不能只补 1-2 这种开放冷门，也要补 0-1 这种早进球后死守路径。
- 不要为了补冷门随意删掉主线最顺比分；主线比分、低比分冷门、卡线比分要同时保留。

## §11 结算口径硬规则（90 分钟 / 异常场次）
- **竞彩所有玩法（HAD/HHAD/比分/总进球/半全场）一律只按 90 分钟（含伤停补时）结算，不含加时和点球。**
- 淘汰赛警惕：“谁晋级” ≠ “90 分钟谁赢”。强队加时/点球过关时 HAD 按 90 分钟可能是“平”，压强队胜/猜含加时比分会误判。
- 预期“90 分钟难分胜负、靠加时点球解决” → 胜平负纳入“平”、比分纳入平局比分，而非单压强队胜。
- 延期/改期：按官方公告（通常限时顺延有效、超时退奖；串关该场常退该场或 SP=1）。腰斩/取消：退奖/退本金，不自行假设。
- 异常场次标“延期/腰斩/待官方裁定”，不提前结算；以竞彩官方公告口径为准。

## §12 玩法体系硬规则
- 玩法含 HAD/HHAD/比分 + **总进球数（TTG）、半全场（HAFU）、串关（M 串 N，含单关）**，均 90 分钟结算（§11）。
- **总进球数**：区间 0/1/2/3/4/5/6/7+；守领先/慢热/破密集差/低比分倾向场次，“0-1 球”或“2 球”是比猜具体比分方差更低的低分出口，优先于纯比分考虑。
- **半全场**：早丢球死守、慢热开局 → 半全场（客/客、平/主、平/客）天然冷门出口。
- **单关**：仅部分场次开放、赔率有官方下限；M 串 N 过关方式与封顶以官方为准，出票前核对（§1）。
- §3 低分玩法类资金用于 TTG/HAFU/比分；先问“哪种玩法表达同一判断方差最低、赔率最优”，不默认买比分。

## §13 行为偏差硬规则
下注前后逐条自检，命中任一即降投或重审：
- 近因/光环偏差：连胜连败、最近一场大比分 ≠ 真实实力，结合对手质量与本届全部样本（§2B）。
- 球星/牌面偏差：牌面 ≠ 状态 ≠ 战意（§2A）。
- 大众盘从众：热门被过度下注压低赔率至无价值，反向评估冷门价值与赔率信号（§2C）。
- 锚定偏差：下注后信息变化要敢推翻原判，不为“已下注”护盘。
- 结果偏差：复盘不只看赢没赢，看下注当时是否正期望/正 CLV（§5）。

## §14 数据时效硬规则
- 每个数据文件标“最后更新时间”，过期即触发置信度降级（见 workflow/evidence-weight.md）。
- 阈值：①首发/伤停/停赛/临场 >24h 过期，停售前再核；②`data/teams`、`group-rank.md` 每轮更新，未含最新一轮或 >7 天过期；③赔率 open/live 都要，live 距停售 >3h 重拉。
- 用过期数据硬下结论 = 违反 §2C；拿不到新数据 → 降额或放弃。

## §15 48 队赛制与最佳第三名战意
- 本届 48 队：12 组、每组前 2 + 8 个最佳第三名 → 32 队进淘汰赛。
- 最佳第三名排序：积分 → 净胜球 → 进球数 →（官方细则：公平竞赛分/抽签）；12 个组第三名拉横向对比表、第 8 名划红线（模板见 `data/group-rank.md`）。
- 战意：某队大概率落小组第 3 时，出线取决于其它组 → **末轮战意高度模糊**；降低胜负预期、增加平/小负/低比分路径、降投或观望。
- 不做自动概率模拟：精确“是否已出线”要等所有组末轮（交错）踢完才定；只做人工横向对比 + 战意定性。
